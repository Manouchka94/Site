Projet de Synthèse

Durée : 8 mois
Équipe : 10 personnes
Rôle : Responsable Technique

Présentation du projet :

Ce projet d'informatique a été réalisé au cours de ma dernière année au sein de la MIAGE de Créteil.
Il était encadré par deux membres du corps enseignant :

- Un consultant fonctionnelle spécialisé dans les méthodes agiles.
- Un second consultant en charge de la partie technique (conception, programmation, infrastructure informatique) .

Méthodologie fonctionnelle et gestion de projet

Pour mener à bien notre projet, nous avons choisi d'utiliser certaines pratiques issues des méthodes agiles :

- Extrem Programming : Technique d'ingéniere logiciel.
Développement et revue de code en binôme
Rédaction de test unitaire
Planning poker

- Scrum : Méthodologie de développement logiciel agile.
Découpage du projet en Sprint
Cycle de développement itératif
Présence d'un Product Owner
Scrum Master à tour de rôle au sein de l'équipe

- TDD : Test Driven Development.
Rédaction des tests avant d'écrire le code.
Exécution des tests via Junit.
Génération automatique des rapports de tests.

- Intégration Continue : Ensemble de pratiques pour assurer une non régression de l'application.
Mise en place d'une plateforme d'intégration continue (Jenkins).
Compilation automatique du code disponible sur le repository.
Livraison automatique de la dernière version de l'application sur la plateforme de démonstration cliente.

Documentation fonctionnelle

À chaque fin de release, nous avons du fournir un certain nombre de documents liés à la gestion du projet.

- Product Backlog.
- Vision Doc.
- Analyse de risque.
- Plan Projet.
- Benchmarking des solutions existantes.
- Diagrammes UML : Use Case Model, Diagramme de Séquence, Diagramme de classe, Diagramme de déploiement, Diagramme de composant.

Réalisations Techniques

Pour répondre au besoin exprimé nous avons choisi de développer un client lourd.
- Le Langage de programmation : Notre application a été développé en Java version 6 .
- Le SGBD : Nous avons opté pour un système de gestion de base de donnée orienté SQL Server.
- Le framework d'accès aux données : Hibernate.

En tant que responsable technique sur ce projet, j'ai dû prendre en charge la gestion des serveurs sous un environnement de virtualisation VMware ESXi.

En collaboration avec mon équipe, j'ai concu l'architecture de notre environnement de développement, de production et de livraison.

Afin que les principes d'intégration continue et de maintien en condition opérationnel soient assurés, il a donc fallu : configurer les différents serveurs, les applications installées et réaliser leurs déploiements et maintenances tout au long du projet.

L'infrastructure du projet

Schéma d'architecture générale :

Pour ce projet, notre infrastructure était composer de quatre serveurs :

- Un serveur d'intégration continue.
(Application d'intégration continue - Jenkins installé sur un Windows Server)

- Une plateforme de démonstration cliente.
(Windows Xp).

- Un serveur de base de donnée.
(Sql Serveur sous Windows Server)

- Un serveur ESB (Entreprise Service Bus).
(ServiceMix par Apache installé sur un Windows Server)
